Basic	What is malware? Name and describe five types.	<b>Malware</b> is any software designed to harm or gain unauthorised access to a computer system.<br><b>Virus:</b> attaches itself to files; activates and replicates when the infected file is opened.<br><b>Worm:</b> self-replicating; spreads across networks by exploiting software vulnerabilities — no user action needed.<br><b>Trojan:</b> disguised as legitimate software; installs harmful payload when run.<br><b>Ransomware:</b> encrypts the victim's files and demands payment for the decryption key.<br><b>Spyware:</b> secretly monitors user activity and sends data (passwords, browsing history) to an attacker. Cloze	Types of malware: a {{c1::virus}} replicates by attaching to files (needs human action to spread). A {{c2::worm}} self-propagates across networks without user interaction. A {{c3::trojan}} is disguised as legitimate software. {{c4::Ransomware}} encrypts files and demands payment. {{c5::Spyware}} covertly monitors user activity. {{c6::Adware}} displays unwanted advertisements. Basic	What is a phishing attack? What is spear phishing?	<b>Phishing:</b> mass fraudulent emails claiming to be from a reputable source (bank, HMRC, PayPal) to trick recipients into revealing passwords, card details, or clicking malicious links that install malware. Relies on creating urgency ("your account will be closed").<br><b>Spear phishing:</b> a targeted phishing attack aimed at a specific individual or organisation, using personal details (name, employer, colleague names) to appear more convincing.	Computer Science::Paper 1::Network Security
Basic	What is a brute force attack? How is it prevented?	A brute force attack systematically tries every possible combination of passwords until the correct one is found. Simple passwords can be cracked in seconds; long complex ones take years.<br><b>Defences:</b> account lockout after N failed attempts, CAPTCHA, rate limiting, long complex passwords (uppercase, lowercase, digits, symbols), two-factor authentication (2FA). Basic	What is a Denial of Service (DoS) attack? What is a DDoS?	<b>DoS:</b> floods a server/network with so many requests that it becomes overloaded and cannot respond to legitimate users — effectively taking a website offline.<br><b>DDoS (Distributed DoS):</b> the same attack launched simultaneously from thousands of infected computers (a <b>botnet</b>) — much harder to block because the traffic comes from many different IP addresses. Botnet owners are often unaware their machines are infected.	Computer Science::Paper 1::Network Security
Basic	What is SQL injection and how does it work?	SQL injection inserts malicious SQL code into an input field (e.g. a login form). If the application does not sanitise input, the injected code is executed by the database.<br><b>Example:</b> entering <code>' OR '1'='1</code> into a password field can bypass authentication by making the SQL query always return true.<br><b>Prevention:</b> input validation/sanitisation, parameterised queries (prepared statements), using stored procedures, applying least-privilege database permissions.	Computer Science::Paper 1::Network Security
Cloze	Network security measures: {{c1::Firewall}} — filters network traffic based on rules. {{c2::Encryption}} — scrambles data so only authorised parties can read it. {{c3::Anti-malware}} software — detects and removes malicious programs. {{c4::Access levels/permissions}} — users only access what they need (principle of least privilege). {{c5::Two-factor authentication (2FA)}} — requires two forms of verification to log in. Basic	What is social engineering? Give three techniques.	Social engineering manipulates people (rather than exploiting technical vulnerabilities) to reveal confidential information or perform insecure actions.<br><b>Techniques:</b><br><b>Phishing:</b> fraudulent emails.<br><b>Pretexting:</b> creating a fabricated scenario (e.g. impersonating IT support) to extract information.<br><b>Baiting:</b> leaving infected USB drives in public places hoping someone will plug one in.<br><b>Tailgating:</b> following an authorised person through a secure door.	Computer Science::Paper 1::Network Security
Basic	What is a firewall and how does it protect a network?	A firewall is a hardware device or software program that monitors and filters all incoming and outgoing network traffic based on a set of predefined security rules.<br><b>Functions:</b> blocks traffic from suspicious IP addresses, blocks certain ports (e.g. port 23 Telnet), inspects packet contents, logs attempted connections.<br><b>Acts as a barrier</b> between a trusted internal network and an untrusted external network (the Internet).	Computer Science::Paper 1::Network Security
Basic	What is a botnet and how is it created?	A botnet is a network of computers infected with malware (called <b>bots</b> or <b>zombies</b>) and secretly controlled remotely by an attacker (the <b>bot herder</b>).<br><b>Created by:</b> distributing malware via phishing emails, drive-by downloads, or infected software. Victims' machines run silently in the background.<br><b>Used for:</b> DDoS attacks, sending spam, credential stuffing, cryptocurrency mining. The device owners are unaware. Basic	What is two-factor authentication (2FA) and why is it more secure than a password alone?	2FA requires two separate forms of verification before granting access. Even if a password is stolen/guessed, the attacker cannot access the account without the second factor.<br><b>Three categories:</b><br>Something you <b>KNOW</b> — password or PIN<br>Something you <b>HAVE</b> — smartphone (SMS code/authenticator app), hardware token<br>Something you <b>ARE</b> — biometric (fingerprint, face recognition)<br>Typical 2FA uses two different categories. Basic	What is penetration testing and why is it used?	Penetration testing (pen testing) is an <b>authorised</b>, simulated cyberattack carried out by professional security experts to identify vulnerabilities in a system before real attackers do.<br>Pen testers have written permission from the organisation. They attempt the same techniques as real attackers (SQL injection, phishing, brute force) and produce a report of vulnerabilities found with recommended fixes.<br>Distinct from hacking — which is unauthorised and illegal under the Computer Misuse Act 1990. Cloze	To prevent SQL injection: use {{c1::parameterised queries}} (prepared statements) so user input is never directly concatenated into SQL. Also: {{c2::input validation}} (reject unexpected characters), {{c3::least-privilege}} database accounts (users cannot DROP tables), and {{c4::web application firewalls}}. Basic	What is HTTPS and how does it differ from HTTP?	HTTPS (HyperText Transfer Protocol Secure) is the secure version of HTTP. Data sent between the browser and server is <b>encrypted</b>, preventing it from being intercepted and read by third parties. Indicated by a padlock icon in the browser. Used for login pages, online banking, and any site handling sensitive data. HTTP sends data in plaintext — anyone intercepting the traffic can read it.	Computer Science::Paper 1::Network Security
Basic	What is data interception and theft?	<b>Data interception</b> (packet sniffing / man-in-the-middle attack) is the unauthorised capturing of data while it is being transmitted across a network. An attacker uses software to intercept unencrypted packets — commonly on unsecured public Wi-Fi.<br><b>Data theft</b> is the stealing of stored data (credentials, personal information, financial records).<br><b>Prevention:</b> encryption (HTTPS/TLS) makes intercepted data unreadable; VPNs encrypt traffic on public networks; avoiding unsecured Wi-Fi; strong access controls on stored data.	Computer Science::Paper 1::Network Security
Basic	What is physical security and how does it protect computer systems?	<b>Physical security</b> protects hardware and data from physical threats: theft, damage, or unauthorised physical access to devices.<br><b>Methods:</b> locked server rooms with key/card or biometric access; CCTV; cable locks on devices; security guards; secure disposal of old storage media (degaussing, shredding HDDs).<br><b>Why it matters:</b> software security is irrelevant if an attacker can physically remove a hard drive or plug in a USB device to copy data directly.	Computer Science::Paper 1::Network Security
Basic	How do strong passwords reduce security risk and what are best practices?	A strong password is long and uses a mix of uppercase, lowercase, digits, and symbols — making it resistant to brute force and dictionary attacks.<br><b>Best practices:</b> unique password per account; never share passwords; change regularly; use a password manager; avoid obvious personal information (name, date of birth).<br><b>Limitation:</b> passwords can still be stolen via phishing or keyloggers, which is why they should be combined with 2FA.	Computer Science::Paper 1::Network Security
Basic	What are user access levels and how do they improve security?	<b>User access levels</b> (permissions/access control) restrict what each user can see, read, write, or execute on a system.<br><b>Principle of least privilege:</b> users are only given the minimum access required for their role. A student can read files but not install software; a teacher can access grades but not payroll data; an administrator has full access.<br><b>Security benefit:</b> if a low-privilege account is compromised, the attacker cannot access confidential data or system files outside that account's permission level.
