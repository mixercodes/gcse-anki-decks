Basic	What is malware? Name and describe five types.	<b>Malware</b> is any software designed to harm or gain unauthorised access to a computer system.<br><b>Virus:</b> attaches itself to files; activates and replicates when the infected file is opened.<br><b>Worm:</b> self-replicating; spreads across networks by exploiting software vulnerabilities — no user action needed.<br><b>Trojan:</b> disguised as legitimate software; installs harmful payload when run.<br><b>Ransomware:</b> encrypts the victim's files and demands payment for the decryption key.<br><b>Spyware:</b> secretly monitors user activity and sends data (passwords, browsing history) to an attacker. Cloze	Types of malware: a {{c1::virus}} replicates by attaching to files (needs human action to spread). A {{c2::worm}} self-propagates across networks without user interaction. A {{c3::trojan}} is disguised as legitimate software. {{c4::Ransomware}} encrypts files and demands payment. {{c5::Spyware}} covertly monitors user activity. {{c6::Adware}} displays unwanted advertisements. Basic	What is a phishing attack? What is spear phishing?	<b>Phishing:</b> mass fraudulent emails claiming to be from a reputable source (bank, HMRC, PayPal) to trick recipients into revealing passwords, card details, or clicking malicious links that install malware. Relies on creating urgency ("your account will be closed").<br><b>Spear phishing:</b> a targeted phishing attack aimed at a specific individual or organisation, using personal details (name, employer, colleague names) to appear more convincing.	Computer Science::Paper 1::Network Security
Basic	What is a brute force attack? How is it prevented?	A brute force attack systematically tries every possible combination of passwords until the correct one is found. Simple passwords can be cracked in seconds; long complex ones take years.<br><b>Defences:</b> account lockout after N failed attempts, CAPTCHA, rate limiting, long complex passwords (uppercase, lowercase, digits, symbols), two-factor authentication (2FA). Basic	What is a Denial of Service (DoS) attack? What is a DDoS?	<b>DoS:</b> floods a server/network with so many requests that it becomes overloaded and cannot respond to legitimate users — effectively taking a website offline.<br><b>DDoS (Distributed DoS):</b> the same attack launched simultaneously from thousands of infected computers (a <b>botnet</b>) — much harder to block because the traffic comes from many different IP addresses. Botnet owners are often unaware their machines are infected.	Computer Science::Paper 1::Network Security
Basic	What is SQL injection and how does it work?	SQL injection inserts malicious SQL code into an input field (e.g. a login form). If the application does not sanitise input, the injected code is executed by the database.<br><b>Example:</b> entering <code>' OR '1'='1</code> into a password field can bypass authentication by making the SQL query always return true.<br><b>Prevention:</b> input validation/sanitisation, parameterised queries (prepared statements), using stored procedures, applying least-privilege database permissions.	Computer Science::Paper 1::Network Security
Cloze	Network security measures: {{c1::Firewall}} — filters network traffic based on rules. {{c2::Encryption}} — scrambles data so only authorised parties can read it. {{c3::Anti-malware}} software — detects and removes malicious programs. {{c4::Access levels/permissions}} — users only access what they need (principle of least privilege). {{c5::Two-factor authentication (2FA)}} — requires two forms of verification to log in. Basic	What is the difference between symmetric and asymmetric encryption?	<b>Symmetric encryption:</b> the same key is used to encrypt and decrypt data. Fast, but the key must be shared securely — if intercepted, data is compromised. Used for bulk data encryption.<br><b>Asymmetric encryption:</b> uses a mathematically linked key pair — a <b>public key</b> (anyone can encrypt with it) and a <b>private key</b> (only the owner can decrypt). Slower but solves the key exchange problem. HTTPS uses asymmetric encryption to securely exchange a symmetric session key.	Computer Science::Paper 1::Network Security
Basic	What is social engineering? Give three techniques.	Social engineering manipulates people (rather than exploiting technical vulnerabilities) to reveal confidential information or perform insecure actions.<br><b>Techniques:</b><br><b>Phishing:</b> fraudulent emails.<br><b>Pretexting:</b> creating a fabricated scenario (e.g. impersonating IT support) to extract information.<br><b>Baiting:</b> leaving infected USB drives in public places hoping someone will plug one in.<br><b>Tailgating:</b> following an authorised person through a secure door. Basic	What is a firewall and how does it protect a network?	A firewall is a hardware device or software program that monitors and filters all incoming and outgoing network traffic based on a set of predefined security rules.<br><b>Functions:</b> blocks traffic from suspicious IP addresses, blocks certain ports (e.g. port 23 Telnet), inspects packet contents, logs attempted connections.<br><b>Acts as a barrier</b> between a trusted internal network and an untrusted external network (the Internet).	Computer Science::Paper 1::Network Security
Basic	What is a botnet and how is it created?	A botnet is a network of computers infected with malware (called <b>bots</b> or <b>zombies</b>) and secretly controlled remotely by an attacker (the <b>bot herder</b>).<br><b>Created by:</b> distributing malware via phishing emails, drive-by downloads, or infected software. Victims' machines run silently in the background.<br><b>Used for:</b> DDoS attacks, sending spam, credential stuffing, cryptocurrency mining. The device owners are unaware.	Computer Science::Paper 1::Network Security
Basic	What is two-factor authentication (2FA) and why is it more secure than a password alone?	2FA requires two separate forms of verification before granting access. Even if a password is stolen/guessed, the attacker cannot access the account without the second factor.<br><b>Three categories:</b><br>Something you <b>KNOW</b> — password or PIN<br>Something you <b>HAVE</b> — smartphone (SMS code/authenticator app), hardware token<br>Something you <b>ARE</b> — biometric (fingerprint, face recognition)<br>Typical 2FA uses two different categories.	Computer Science::Paper 1::Network Security
Basic	What is penetration testing and why is it used?	Penetration testing (pen testing) is an <b>authorised</b>, simulated cyberattack carried out by professional security experts to identify vulnerabilities in a system before real attackers do.<br>Pen testers have written permission from the organisation. They attempt the same techniques as real attackers (SQL injection, phishing, brute force) and produce a report of vulnerabilities found with recommended fixes.<br>Distinct from hacking — which is unauthorised and illegal under the Computer Misuse Act 1990. Cloze	To prevent SQL injection: use {{c1::parameterised queries}} (prepared statements) so user input is never directly concatenated into SQL. Also: {{c2::input validation}} (reject unexpected characters), {{c3::least-privilege}} database accounts (users cannot DROP tables), and {{c4::web application firewalls}}. Basic	What encryption does HTTPS use and how does it work?	HTTPS uses <b>TLS (Transport Layer Security)</b>. The process (TLS handshake):<br>1. Server sends its <b>public key</b> and SSL certificate (verified by a trusted Certificate Authority).<br>2. Browser verifies the certificate.<br>3. Browser uses the server's public key to encrypt a symmetric session key and send it.<br>4. Server decrypts it with its private key.<br>5. Both sides now share the same symmetric session key and use it for fast encrypted communication.<br>Padlock icon = HTTPS in use.	Computer Science::Paper 1::Network Security
