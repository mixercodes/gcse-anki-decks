Basic	What is input validation? Name and describe five types of validation check.	Input validation ensures data is sensible and acceptable before the program processes it.<br><b>1. Range check:</b> is the value within acceptable limits? (age 0–120)<br><b>2. Type check:</b> is the data the correct type? (must be an integer, not letters)<br><b>3. Length check:</b> is the data the right length? (password 8+ characters)<br><b>4. Presence check:</b> has a required field been filled in?<br><b>5. Format check:</b> does the data match a required pattern? (email contains @, date is DD/MM/YYYY)	Computer Science::Paper 2::Producing Robust Programs

Basic	What is the difference between a syntax error and a logic error?	<b>Syntax error:</b> code breaks the grammatical rules of the language — the program <i>cannot run at all</i>. Found by the compiler/interpreter before execution.<br>Examples: missing colon, misspelled keyword (<code>prit</code> instead of <code>print</code>), unclosed bracket.<br><b>Logic error:</b> code runs without crashing but produces <i>wrong results</i> — the algorithm is flawed.<br>Examples: using &lt; instead of &lt;=, off-by-one error in a loop (iterating one too many or few times), wrong formula. Basic	What is a runtime error? Give two examples.	A runtime error occurs <i>during</i> program execution and causes the program to crash. Not detected at compile time.<br><b>Examples:</b><br>• Division by zero: <code>x = 10 / 0</code><br>• Index out of bounds: accessing <code>array[10]</code> when the array only has 5 elements<br>• Trying to open a file that does not exist<br>• Stack overflow from infinite recursion<br>• Type mismatch at runtime	Computer Science::Paper 2::Producing Robust Programs

Basic	Why do we test with boundary values? What is a boundary test for age 1–120?	Boundary values are the most likely places for logic errors — conditions using &lt;, &gt;, &lt;=, &gt;= are very commonly wrong by one.<br><b>For valid range 1–120:</b><br>Boundary values: 1 (minimum valid), 120 (maximum valid)<br>Just outside: 0 (should be rejected), 121 (should be rejected)<br>All four should be tested. A bug using &lt; instead of &lt;= would pass 119 but incorrectly reject 120.	Computer Science::Paper 2::Producing Robust Programs
Basic	What techniques make a program more maintainable?	<b>1. Meaningful identifiers:</b> <code>playerScore</code> not <code>x</code><br><b>2. Comments:</b> explain the WHY (non-obvious logic), not the WHAT<br><b>3. Indentation:</b> consistent indentation makes structure visible<br><b>4. Modular design:</b> break code into subroutines (functions/procedures)<br><b>5. Constants:</b> use named constants instead of "magic numbers" (<code>MAX_LIVES = 3</code> not <code>3</code>)<br><b>6. Consistent style:</b> naming conventions (camelCase or snake_case consistently)	Computer Science::Paper 2::Producing Robust Programs
Basic	What is authentication and why is it used in programs?	Authentication confirms the identity of a user before granting access to a system or program.<br><b>Simple implementation:</b> the user enters a username and password; the program checks these against stored credentials and only allows access if they match.<br><b>Purpose:</b> prevents unauthorised access to data or functionality.<br><b>Good practice:</b> limit failed login attempts to slow down brute force attacks; use strong password requirements. Basic	What makes a program 'robust'?	A robust program handles all inputs and situations gracefully without crashing:<br>• <b>Input validation</b> — rejects invalid data before processing<br>• <b>Error handling</b> — gracefully manages runtime errors (try/except/catch blocks)<br>• <b>Authentication</b> — controls who can access the program<br>• <b>Testing</b> with normal, boundary and erroneous test data<br>• <b>Meaningful error messages</b> — tells the user what went wrong and how to fix it	Computer Science::Paper 2::Producing Robust Programs
