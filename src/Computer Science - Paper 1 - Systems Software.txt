Computer Science::Paper 1::Systems Software
Basic	What is an Operating System (OS) and what are its five main roles?	The OS is system software that manages hardware and provides an environment for application software to run.<br><b>1. Memory management</b> — allocates RAM to programs, manages virtual memory.<br><b>2. Process management</b> — schedules which process gets CPU time (multitasking).<br><b>3. File management</b> — organises files in directories, controls read/write access.<br><b>4. Device management</b> — uses device drivers to communicate with hardware.<br><b>5. User interface</b> — provides GUI (Windows, macOS) or CLI (Linux terminal).	Computer Science::Paper 1::Systems Software
Basic	What are utility programs? Give five examples.	Utility programs are software that perform system maintenance to keep the computer running efficiently.<br><b>Examples:</b><br>• <b>Disk defragmenter</b> — reorganises fragmented files on HDDs for faster access<br>• <b>Anti-malware</b> — scans and removes malicious software<br>• <b>Backup software</b> — creates copies of data for recovery<br>• <b>Disk cleanup</b> — removes temporary/unnecessary files to free space<br>• <b>Encryption tools</b> — secures files with encryption<br>• <b>File compression tools</b> — reduce file sizes (ZIP)	Computer Science::Paper 1::Systems Software
Basic	What is the difference between a compiler and an interpreter?	<b>Compiler:</b> translates the entire source code into machine code <i>all at once</i>, producing a standalone executable file (.exe). Errors only reported after full translation. Fast to run after compilation. Code distributed as executable (source hidden).<br><b>Interpreter:</b> translates and executes source code <i>one line at a time</i>. Stops at the first error — easier to debug. No executable produced. Slower to run. Source code must be present on the machine.	Computer Science::Paper 1::Systems Software

Basic	What is an assembler? How does it differ from a compiler?	An assembler translates <b>assembly language</b> (low-level mnemonics: ADD, MOV, LDA, STA) into <b>machine code</b> (binary). Each assembly instruction typically translates to <b>one</b> machine code instruction (unlike a compiler where one high-level statement may produce many machine code instructions). Assembly language is processor-specific — different CPU architectures have different instruction sets.	Computer Science::Paper 1::Systems Software
Basic	What is a device driver and why is it needed?	A device driver is a small program that allows the OS to communicate with a specific piece of hardware (printer, keyboard, GPU, webcam). It translates the OS's generic commands into hardware-specific instructions the device understands.<br><b>Without drivers:</b> the OS cannot use the hardware. When you plug in a new device, the OS looks for the appropriate driver (often downloaded automatically or from a disc).	Computer Science::Paper 1::Systems Software
Basic	What is defragmentation and when is it needed?	Defragmentation is a utility program that reorganises the data on an HDD so that each file is stored in <b>contiguous</b> (adjacent) sectors on the disk. Over time, as files are created, deleted, and modified, they become <b>fragmented</b> (split across non-adjacent sectors), causing the read head to travel more — slower access.<br><b>NOT needed</b> for SSDs — they have no moving read heads, access all sectors equally fast, and repeated defragmentation would wear them out.	Computer Science::Paper 1::Systems Software
Basic	What is multitasking and how does the OS achieve it?	Multitasking is the ability of an OS to run multiple programs apparently simultaneously on a single-core CPU. The OS <b>scheduler</b> rapidly switches between processes, allocating short time slices to each. Since switching is so fast (milliseconds), programs appear to run simultaneously.<br>With multiple cores, true parallel execution is possible for different processes/threads. Basic	What is the difference between a CLI and a GUI?	<b>CLI (Command Line Interface):</b> user types text commands. Advantages: fast, powerful, scriptable, low resource use. Disadvantages: steep learning curve, must memorise commands. Used by: programmers, system administrators.<br><b>GUI (Graphical User Interface):</b> user interacts with windows, icons, menus, pointer (WIMP). Advantages: intuitive, no commands to memorise, accessible. Disadvantages: more CPU/RAM required, slower for some tasks.	Computer Science::Paper 1::Systems Software
