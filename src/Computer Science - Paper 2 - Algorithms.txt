Basic	What is abstraction? Give a real-world example.	Abstraction means ignoring irrelevant details and representing only what matters for the task at hand.<br><b>Example 1:</b> A London Underground map shows stations and connections — it ignores actual scale, curves, depths underground, and above-ground geography. This makes it useful for navigation.<br><b>Example 2:</b> A programmer using <code>print()</code> doesn't need to know how the OS and hardware output text to screen — that complexity is hidden (abstracted away). Basic	Describe the linear search algorithm. When is it appropriate?	<b>Linear search</b> checks each element in the list one by one from the start until the target is found or the list is exhausted.<br><b>Steps:</b> start at index 0; compare current element to target; if match → found; if not → move to next; if end reached → not found.<br><b>Time complexity:</b> O(n) — worst case checks every element.<br><b>Appropriate when:</b> data is unsorted, list is small, or only one search is needed. Basic	Describe the binary search algorithm. What is its requirement?	<b>Requirement: data MUST be sorted.</b><br><b>Steps:</b><br>1. Find the middle element.<br>2. If middle = target → found.<br>3. If target &lt; middle → discard right half, repeat on left half.<br>4. If target &gt; middle → discard left half, repeat on right half.<br>5. Repeat until found or list empty → not found.<br><b>Time complexity:</b> O(log n) — each step halves the search space. For 1,024 elements: max 10 comparisons (log₂1024 = 10). Basic	Describe the bubble sort algorithm with a worked example.	<b>Algorithm:</b> repeatedly pass through the list; compare adjacent pairs; swap if in wrong order; repeat until a full pass with no swaps occurs.<br><b>Example:</b> sort [5, 3, 8, 1]<br>Pass 1: [3,5,8,1]→[3,5,8,1]→[3,5,1,8] (8 and 1 swapped)<br>Pass 2: [3,5,1,8]→[3,1,5,8]→[3,1,5,8]<br>Pass 3: [1,3,5,8]→[1,3,5,8] (no swaps — sorted!)<br><b>Complexity:</b> O(n²) worst case; O(n) best case (already sorted).	Computer Science::Paper 2::Algorithms
Basic	Describe the merge sort algorithm.	<b>Merge sort uses divide and conquer:</b><br>1. <b>Divide:</b> repeatedly split the list in half until every sublist contains only one element.<br>2. <b>Conquer:</b> merge pairs of sublists back together in sorted order by comparing elements one by one.<br>3. Continue merging until the full list is reconstructed in sorted order.<br><b>Example:</b> [5,3,8,1] → [5,3] [8,1] → [5][3][8][1] → [3,5][1,8] → [1,3,5,8]<br><b>Complexity:</b> O(n log n) — much faster than bubble for large datasets but uses more memory.	Computer Science::Paper 2::Algorithms
Basic	Describe the insertion sort algorithm.	<b>Insertion sort</b> builds a sorted portion of the list one element at a time:<br>1. Start with the second element; compare it to the sorted portion.<br>2. Shift larger elements right to make space.<br>3. Insert the element in the correct position.<br>4. Repeat for each remaining element.<br><b>Example:</b> [5,3,8,1] → [3,5,8,1] → [3,5,8,1] → [1,3,5,8]<br><b>Complexity:</b> O(n) best case (nearly sorted); O(n²) worst case. Efficient for small or nearly-sorted lists.	Computer Science::Paper 2::Algorithms
Cloze	Sorting algorithm comparison: Bubble sort — simple, O({{c1::n²}}) worst case, in-place. Merge sort — O({{c2::n log n}}), requires extra {{c3::memory}} for merging. Insertion sort — efficient for nearly-{{c4::sorted}} data, O({{c5::n}}) best case, O(n²) worst case. For large datasets, {{c6::merge sort}} is most efficient.		Computer Science::Paper 2::Algorithms
