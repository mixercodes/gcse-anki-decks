The processor can only understand machine code — it cannot directly execute high-level language or assembly code.	Computer Science::Paper 2::Languages and IDEs

Basic	What is the difference between a compiler and an interpreter in terms of error reporting?	<b>Compiler:</b> Reports all errors after translating the entire program. The program does not run until all errors are fixed.<br><b>Interpreter:</b> Stops and reports an error as soon as it encounters one, executing line by line. This makes it easier to find and fix errors one at a time.	Computer Science::Paper 2::Languages and IDEs
Basic	What does a translator do? Name the three types of translator.	A translator converts source code into machine code (binary) so the CPU can execute it.<br><b>Three types:</b><br>1. <b>Compiler</b> — translates all code at once into an executable file<br>2. <b>Interpreter</b> — translates and executes one line at a time<br>3. <b>Assembler</b> — translates assembly language into machine code	Computer Science::Paper 2::Languages and IDEs
Basic	What is syntax highlighting in an IDE and why is it useful?	Syntax highlighting automatically colours different parts of code (keywords, strings, comments, variables) using different colours.<br><b>Why useful:</b> makes code easier to read; helps spot typos (a misspelled keyword won't be highlighted in the correct colour); distinguishes comments from executable code. Basic	Why must high-level language programs be translated before the CPU can run them?	The CPU can only execute <b>machine code</b> (binary instructions). High-level languages like Python use English-like syntax that the CPU does not understand. A compiler or interpreter must convert the high-level source code into machine code first.
