Basic	What are the advantages and disadvantages of high-level vs low-level programming languages?	<b>High-level (Python, Java, C#):</b><br>Advantages: human-readable, easier to write/debug, portable (runs on different hardware), abstracted from hardware details, large standard libraries.<br>Disadvantages: translated (slower execution), less direct hardware control.<br><b>Low-level (Assembly, Machine Code):</b><br>Advantages: directly controls hardware, very fast execution, small memory footprint.<br>Disadvantages: hardware-specific (not portable), difficult to read/write/debug, time-consuming. Basic	What is an IDE (Integrated Development Environment)? Why use one?	An IDE is a software application that provides a complete environment for writing, testing, and debugging programs. It combines multiple development tools into one interface.<br><b>Why use an IDE?</b> Increases productivity — tools like syntax highlighting, auto-completion, and integrated debugging remove friction and help find errors faster than using a plain text editor and separate tools. Basic	What does syntax highlighting do and why is it helpful?	Syntax highlighting colours different parts of code (keywords, strings, numbers, comments) using different colours automatically.<br><b>Benefits:</b> makes code easier to read at a glance; helps spot typos (a misspelled keyword won't be highlighted); distinguishes comments from executable code; reduces cognitive load when reading complex code. Basic	What is the difference between a compiler and an interpreter? Which is used in IDEs?	<b>Compiler:</b> translates entire source code to machine code at once → creates an executable. Errors reported after full translation. Fast execution. Used for production software (C, C++).<br><b>Interpreter:</b> translates and executes one line at a time. Stops at first error. No separate executable. Slower. Used for development/scripting (Python, JavaScript).<br><b>IDEs</b> typically support both — Python IDEs use interpreters; C++ IDEs use compilers.
