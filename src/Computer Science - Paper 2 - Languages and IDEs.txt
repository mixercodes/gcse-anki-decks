Basic	What are the advantages and disadvantages of high-level vs low-level programming languages?	<b>High-level (Python, Java, C#):</b><br>Advantages: human-readable, easier to write/debug, portable (runs on different hardware), abstracted from hardware details, large standard libraries.<br>Disadvantages: translated (slower execution), less direct hardware control.<br><b>Low-level (Assembly, Machine Code):</b><br>Advantages: directly controls hardware, very fast execution, small memory footprint.<br>Disadvantages: hardware-specific (not portable), difficult to read/write/debug, time-consuming. Basic	What is an IDE (Integrated Development Environment)? Why use one?	An IDE is a software application that provides a complete environment for writing, testing, and debugging programs. It combines multiple development tools into one interface.<br><b>Why use an IDE?</b> Increases productivity — tools like syntax highlighting, auto-completion, and integrated debugging remove friction and help find errors faster than using a plain text editor and separate tools. Basic	What is a breakpoint and how is it used for debugging?	A breakpoint is a marker placed on a specific line of code in an IDE. When the program runs, it <b>pauses execution</b> at that line — allowing the developer to:<br>• Inspect the current values of all variables<br>• Check that the program has reached that point<br>• Step through subsequent lines one at a time<br>• Identify exactly where the logic diverges from expectation<br>Breakpoints are used to debug logic errors — code that runs but gives wrong results. Basic	Explain: step-over, step-into, and step-out in IDE debugging.	<b>Step-over:</b> executes the current line and pauses at the next line — if the line is a function call, the whole function runs without entering it.<br><b>Step-into:</b> if the current line calls a function, execution enters the function body line by line — useful for debugging subroutines.<br><b>Step-out:</b> finishes executing the current function and pauses at the line after the call — used when you've seen enough of a subroutine. Basic	What does syntax highlighting do and why is it helpful?	Syntax highlighting colours different parts of code (keywords, strings, numbers, comments) using different colours automatically.<br><b>Benefits:</b> makes code easier to read at a glance; helps spot typos (a misspelled keyword won't be highlighted); distinguishes comments from executable code; reduces cognitive load when reading complex code. Basic	What is the difference between a compiler and an interpreter? Which is used in IDEs?	<b>Compiler:</b> translates entire source code to machine code at once → creates an executable. Errors reported after full translation. Fast execution. Used for production software (C, C++).<br><b>Interpreter:</b> translates and executes one line at a time. Stops at first error. No separate executable. Slower. Used for development/scripting (Python, JavaScript).<br><b>IDEs</b> typically support both — Python IDEs use interpreters; C++ IDEs use compilers. Basic	What is auto-complete (IntelliSense) in an IDE?	Auto-complete suggests completions for partially typed code — variable names, function names, method parameters, module contents. <br><b>Benefits:</b> reduces typing errors (typos in identifiers), speeds up coding, helps discover available functions/methods without consulting documentation, reminds programmer of required parameters/argument types.
