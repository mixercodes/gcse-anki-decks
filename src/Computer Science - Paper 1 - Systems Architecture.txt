Basic	What does CPU stand for and what is its purpose?	The <b>Central Processing Unit</b> is the brain of the computer. It processes all instructions by carrying out the <b>Fetch-Decode-Execute (FDE) cycle</b> repeatedly. It performs arithmetic, logic, and control operations on data.	Computer Science::Paper 1::Systems Architecture
Cloze	The four main components of the CPU are: {{c1::ALU}} (performs arithmetic and logical operations), {{c2::Control Unit (CU)}} (coordinates all CPU activities, manages the FDE cycle), {{c3::Cache}} (fast temporary storage for frequently used data/instructions), and {{c4::registers}} (tiny, extremely fast storage locations inside the CPU itself).		Computer Science::Paper 1::Systems Architecture
Basic	What are the three stages of the Fetch-Decode-Execute (FDE) cycle?	<b>1. FETCH:</b> The CU copies the address in the Program Counter (PC) to the MAR. The instruction is fetched from RAM into the MDR, then copied to the Current Instruction Register. The PC increments by 1 to point to the next instruction.<br><b>2. DECODE:</b> The CU decodes the instruction — determines what operation is required and what operands are needed.<br><b>3. EXECUTE:</b> The ALU or other components carry out the instruction (arithmetic, data movement, branching, I/O).	Computer Science::Paper 1::Systems Architecture
Cloze	Key CPU registers: {{c1::Program Counter (PC)}} holds the address of the <i>next</i> instruction. {{c2::Accumulator (ACC)}} stores results of ALU operations. {{c3::Memory Address Register (MAR)}} holds the address being read from or written to. {{c4::Memory Data Register (MDR)}} holds data being transferred to/from memory. {{c5::Current Instruction Register (CIR)}} holds the instruction currently being decoded/executed.		Computer Science::Paper 1::Systems Architecture
Cloze	The fetch stage step-by-step: address in {{c1::PC}} → copied to {{c2::MAR}} → instruction fetched from {{c3::RAM}} into {{c4::MDR}} → copied to {{c5::CIR}} → PC {{c6::incremented}} by 1 ready for the next instruction.		Computer Science::Paper 1::Systems Architecture
Basic	What three factors affect CPU performance? Explain each.	<b>1. Clock speed (GHz):</b> Higher = more FDE cycles per second. A 3 GHz CPU completes 3 billion cycles/second.<br><b>2. Number of cores:</b> Each core processes an independent instruction stream. Dual-core can process two threads simultaneously — better for multi-tasking and parallel programs.<br><b>3. Cache size:</b> Larger cache = more data kept close to the CPU, reducing slow RAM accesses. L1 (fastest/smallest) → L2 → L3 (largest/slowest but still far faster than RAM).	Computer Science::Paper 1::Systems Architecture
Cloze	Von Neumann architecture stores both {{c1::programs}} and {{c2::data}} in the same {{c3::memory}}. The CPU connects to memory and I/O via three buses: {{c4::data bus}} (carries data), {{c5::address bus}} (carries memory addresses — one directional), {{c6::control bus}} (carries control signals like read/write). Basic	What is an embedded system? Give three examples.	An embedded system is a computer system with a fixed program built into another device to perform a specific, dedicated function. The programming rarely or never changes.<br><b>Examples:</b> microcontroller in a washing machine (controls cycles), ABS braking system in a car, processor in a pacemaker, CPU in a traffic light, microcontroller in a microwave, smart TV processor.	Computer Science::Paper 1::Systems Architecture
Basic	What is cache memory? Describe the three cache levels.	Cache is extremely fast SRAM built into or adjacent to the CPU chip. It stores recently/frequently used instructions and data to reduce the number of slower main RAM accesses.<br><b>L1:</b> Smallest (KB range), fastest, built directly into each core.<br><b>L2:</b> Larger (MB range), slightly slower, on or near the CPU die.<br><b>L3:</b> Largest (several MB), shared between all cores, slowest cache but still much faster than RAM.	Computer Science::Paper 1::Systems Architecture
Basic	What is the difference between increasing clock speed and adding more cores?	<b>Higher clock speed:</b> each core completes more FDE cycles per second — benefits single-threaded tasks most.<br><b>More cores:</b> multiple instruction streams execute in true parallel — benefits multi-threaded tasks (video editing, running multiple programs simultaneously).<br><b>Key point:</b> Not all programs can exploit multiple cores (some tasks are sequential), so more cores does not always give proportional speed improvement.	Computer Science::Paper 1::Systems Architecture
Basic	What is the difference between primary storage and secondary storage?	<b>Primary storage</b> (RAM, ROM, Cache): directly accessible by the CPU; fast; limited capacity; volatile (RAM) or non-volatile (ROM/Cache).<br><b>Secondary storage</b> (HDD, SSD, optical): not directly accessed by CPU; slower; large/permanent storage; always non-volatile. Data must be loaded into RAM before the CPU can use it.	Computer Science::Paper 1::Systems Architecture
Basic	What are the advantages of having multiple CPU cores?	Multiple cores allow <b>parallel processing</b> — each core can execute a different thread simultaneously. Benefits: faster multi-tasking, better performance in multi-threaded applications (games, video editing, databases). Modern CPUs have 4–32+ cores. However, performance gain depends on how well the software is written to use multiple threads.	Computer Science::Paper 1::Systems Architecture
